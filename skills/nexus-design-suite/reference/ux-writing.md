# UX Writing

## Goals
- Make the next action, consequence, and system state obvious.

## Guidelines
- Use direct language and active voice.
- Tell users what happened and what they can do next.
- Match tone to context: calm for risk, light for routine, precise for irreversible actions.
- Prefer specific labels over clever ones.

## Avoid
- Jargon without explanation.
- Generic confirmation text with no user value.
- Overly friendly error messages that hide the fix.
