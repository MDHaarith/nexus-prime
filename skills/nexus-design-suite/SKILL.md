# Nexus Design Suite

Nexus Design Suite is the productized design layer for Nexus-Prime. It folds the assimilated design skill, 17 steering commands, and curated anti-patterns into one Nexus-native surface.

## Use This Skill When
- Building or redesigning web UI, product surfaces, marketing pages, dashboards, onboarding, or component systems.
- Reviewing an existing interface for clarity, consistency, accessibility, responsiveness, or motion quality.
- Steering a UI toward a stronger visual point-of-view instead of default AI aesthetics.

## Workflow
1. Set a clear design direction before touching implementation.
2. Use design tokens and reusable components before one-off styling.
3. Match the command to the problem:
   - `audit`, `critique`, `normalize`, `polish`
   - `clarify`, `distill`, `harden`, `optimize`
   - `animate`, `colorize`, `bolder`, `quieter`, `delight`
   - `adapt`, `extract`, `onboard`, `teach`
4. Validate across accessibility, performance, and responsive breakpoints before handoff.

## Command Pack
- `design.adapt`: Rethink the same experience for another device or context.
- `design.animate`: Add motion that communicates state and hierarchy.
- `design.audit`: Produce a prioritized quality report without hiding defects.
- `design.bolder`: Push visual energy and distinctiveness.
- `design.clarify`: Rewrite interface copy so intent is obvious.
- `design.colorize`: Introduce strategic color and tonal contrast.
- `design.critique`: Evaluate whether the experience actually works as design.
- `design.delight`: Add tasteful surprise and personality.
- `design.distill`: Remove visual and structural excess.
- `design.extract`: Pull patterns into reusable components and tokens.
- `design.harden`: Prepare for edge cases, errors, i18n, and overflow.
- `design.normalize`: Bring a feature back to system standards.
- `design.onboard`: Improve first-run and getting-started flows.
- `design.optimize`: Reduce user-perceived performance cost.
- `design.polish`: Final visual and interaction pass.
- `design.quieter`: Reduce unnecessary visual aggression.
- `design.teach`: Capture persistent design context for the project.

## Anti-Patterns
- Do not default to interchangeable AI-generated layouts.
- Do not use overplayed type stacks such as Inter, Arial, or generic system-only solutions unless the product already does.
- Do not hide weak hierarchy behind gradients, glass cards, or decorative shadows.
- Do not use gray text on saturated backgrounds or low-contrast status colors.
- Do not animate layout properties when opacity or transform will do.
- Do not treat mobile as a scaled-down desktop.
